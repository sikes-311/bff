# Architecture Decision Records (ADR)

このディレクトリにはプロジェクトの重要なアーキテクチャ上の意思決定を記録しています。

## ADRとは

ADR (Architecture Decision Record) は、アーキテクチャ上の重要な決定をその背景・理由・影響とともに記録するドキュメントです。新しいメンバーや将来の自分が「なぜこの設計になったのか」を理解できるようにします。

## ステータスの定義

| ステータス | 意味 |
|---|---|
| `提案中` | 議論・検討中の決定 |
| `採用` | 現在有効な決定 |
| `非推奨` | かつて採用されたが現在は推奨しない |
| `廃止` | 別のADRに置き換えられた |

## 新しいADRの追加方法

1. 次の番号のファイルを `docs/adr/XXXX-title.md` として作成
2. 以下のテンプレートを使用:

```markdown
# ADR-XXXX: タイトル

- **ステータス**: 提案中 / 採用 / 非推奨 / 廃止
- **決定日**: YYYY-MM-DD
- **決定者**: （名前またはチーム名）
- **関連ADR**: ADR-XXXX

## 背景・課題

なぜこの決定が必要になったか。解決したい問題や制約を記述。

## 決定内容

何を決定したか。

## 理由

この決定を選んだ具体的な理由。

## 検討した代替案

| 案 | 概要 | 却下理由 |
|---|---|---|

## 結果・影響

### メリット
### デメリット・トレードオフ
### 対応が必要なこと
```
